## 1. 전체 메뉴 구조

```
문제은행
├── 문제지 만들기      ← 핵심 기능
```

## 2. 문제지 만들기 — 3단계 위저드

### Step 1: 구성하기

#### 기본 정보
- 과목, 출판사 → **외부 API**로 목록 제공

#### 문제지 종류 선택 (분기점)

| 종류 | 설명 |
|------|------|
| 간편형 | 주어진 구성 안에서 간단하게 선택 |
| 고급형 (맞춤형) | 범위, 난이도, 문항 수 등 세부 속성 직접 지정 |

> 종류 변경 시 confirm 모달: "문제지 종류를 변경하면 현재 설정이 초기화됩니다."

---

#### 간편형 - 문제지 유형

| 유형 | 설명 | 난이도 선택 |
|------|------|------------|
| 표준 문제지 | 소단원 기준 필수 문제 | 없음 |
| 수준별 문제지 | 상/중/하 난이도 | 상·중·하 단일 선택 |
| 최상위 문제지 | 블라박스 수준 최상위 | 없음 (자동) |

**단원 선택**: 트리 구조 (대단원 > 중단원 > 소단원), 복수 선택, 외부 API 데이터

**문항 수**: 5 / 10 / 15 고정 선택

**규칙**: 수준별 선택 시 난이도 필드 추가 노출 → 직접 입력 금지, 난이도 선택으로만 구성

---

#### 고급형 — 3컬럼 레이아웃

**컬럼 1: 단원 선택** (간편형과 동일한 트리 구조)

**컬럼 2: 부가 필터** (선택, 기본값 전체 선택)

| 필터 | 옵션 |
|------|------|
| 평가 영역 | 이해력 / 계산력 / 문제해결력 / 추론력 (복수) |
| 문제 형태 | 객관식 / 주관식 (복수) |
| 기출 문제 | 포함 / 제외 |
| 이전 출제 문제 | 포함 / 제외 |

**컬럼 3: 문항 수 및 난이도** (필수)
- 최상 / 상 / 중 / 하 각각 출제 수 직접 입력
- 우측에 선택 범위 내 출제 가능한 최대 수 표시 (외부 API 반환값)
- 단원/필터 변경 시 최대 수 재조회 필요

---

### Step 2: 편집하기

- 외부 API에서 가져온 문항을 좌우 분할 화면으로 표시
  - 좌: 문항 미리보기
  - 우: 문항 목록 (번호 / 난이도 / 문제형태 / 단원명)
- 문항 개별 제거 가능
- 문항 순서 변경 가능
- 뒤로 가기 시 confirm: "현재 편집 중인 문항 리스트가 초기화됩니다."

---

### Step 3: 완성하기

| 항목 | 내용 |
|------|------|
| 문제지 이름 | 직접 입력 |
| 레이아웃 | 4장형 / 6장형 / 8장형 |
| 선생님 | 자동 (로그인 정보) |
| 출력자 | 직접 입력 |
| 출력일 | 날짜 선택 |

**저장 방식** (기존 문제지 수정 시 confirm):
- 덮어쓰기
- 새로 저장

## 4. 외부 API 연동 포인트

| 호출 시점 | API 역할 |
|----------|----------|
| 기본 정보 진입 시 | 과목 목록 조회 |
| 과목 선택 후 | 출판사 목록 조회 |
| 과목+출판사 선택 후 | 단원 트리 조회 |
| 단원+조건 선택 시마다 | 난이도별 가용 문항 수 조회 (고급형) |
| 다음 단계(편집하기) 진입 시 | 문항 JSON 조회 (실제 문항 데이터) |

## 5. 백엔드 설계 요약

### API 흐름

```
[외부 API]
  과목/출판사/단원 트리 → 화면 구성용
  문항 JSON 조회       → 조건(단원+유형+문항수+난이도+필터) 기반

[내부 서버]
  문제지 저장 (신규/덮어쓰기)
  문제지 보관함 목록 조회/필터
  PDF 생성 (레이아웃 타입별)
```

### 문항 조회 파라미터

**간편형**
```
단원 ID 목록 + 문제지 유형 + 문항 수 + 난이도(수준별 시)
```

**고급형**
```
단원 ID 목록
+ 평가 영역 필터 (복수)
+ 문제 형태 필터 (복수)
+ 기출 포함 여부
+ 이전 출제 제외 여부
+ 난이도별 요청 수 { 최상: N, 상: N, 중: N, 하: N }
```

## 6. DB 스키마

파일: `sql/worksheet_ddl.sql`

### 테이블 구성 (5개)

```
todo_common_code    공통 코드 마스터
todo_worksheet      문제지 마스터
  ├── todo_ws_unit      선택 단원 목록 (1:N)
  ├── todo_ws_qst       문항 목록 (1:N)
  └── todo_ws_filter    고급형 부가 필터 (1:N, EAV 패턴)
```

### 핵심 설계 판단

1. **난이도별 문항 수 비정규화** — `todo_worksheet`에 `diff_top_cnt`, `diff_high_cnt`, `diff_mid_cnt`, `diff_low_cnt` 직접 저장
   → 보관함 목록 조회 시 JOIN 없이 난이도 분포 표시 가능

2. **부가 필터 EAV 패턴** — `todo_ws_filter`에 `filter_type` + `filter_val` 조합
   → 평가영역 4가지 + 문제형태 + 기출/이전출제를 단일 테이블로 처리
   → 향후 필터 추가 시 스키마 변경 불필요

3. **FK 비활성화 운영 검토 권장** — INSERT/DELETE 성능 고려

### 공통 코드 그룹

| code_group | 설명 | 코드값 |
|-----------|------|--------|
| WS_TYPE | 문제지 종류 | SIMPLE / ADVANCED |
| WS_CLS | 문제지 유형 | STD / LEVEL / TOP |
| DIFFICULTY | 난이도 | TOP / HIGH / MID / LOW |
| LAYOUT | 레이아웃 | L4 / L6 / L8 |
| WS_STATUS | 문제지 상태 | COMPLETE |
| QST_TYPE | 문제 형태 | OBJ / SUBJ |
| EVAL_AREA | 평가 영역 | COMPR / CALC / PROB / REASON |
| PAST_QST | 기출 필터 | INCLUDE / EXCLUDE |
| PREV_QST | 이전 출제 필터 | INCLUDE / EXCLUDE |

## 7. 미결 사항 (기획 메모)

- 최상위 문제지 우측 안내 문구 미작성 ("최상위 학습에 맞는 설명 필요함")
- 중 1-2 학기 정보 선택 UI 필요 여부 검토 ("중 1-2 선택 열어야 할지, 학기 정보 필요한지")
- 소단원 기준 문제지 보관함 노출 방식 확인 필요
