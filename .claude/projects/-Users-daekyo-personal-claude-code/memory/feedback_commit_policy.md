name: feedback_commit_policy
description: 커밋/푸쉬는 명시적 요청 시에만 실행
type: feedback
---

커밋 및 푸쉬는 사용자가 명시적으로 요청할 때만 실행한다. 코드 변경, 파일 수정, TASKS.md 업데이트 후 자동으로 커밋/푸쉬하지 않는다.

**Why:** 사용자가 준비되기 전에 커밋/푸쉬되는 것을 원하지 않음.

**How to apply:** 작업 완료 후 "커밋할까요?"라고 묻지도 말고, 그냥 작업 결과만 보고한다. 사용자가 "/commit", "커밋해줘", "푸쉬해줘" 등을 명시적으로 말할 때만 실행한다.
